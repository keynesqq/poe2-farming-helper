## 简介

**POE2 开荒任务清单**是一款专为《流放之路2》（Path of Exile 2）设计的桌面辅助工具。它帮助玩家在开荒过程中追踪各章节的地图任务、奖励和完成进度，支持浮窗模式、地图自动检测、一键清查未完成的奖励任务等功能。

### 主要功能

- **任务进度追踪**：勾选已完成的任务，实时查看总进度
- **地图攻略查看**：每个地图都有详细的探索笔记和任务说明
- **奖励任务高亮**：带有奖励标签的任务会特殊显示，便于识别
- **一键清查**：快速查看所有未完成的奖励任务
- **浮窗模式**：游戏内悬浮窗口显示当前地图任务
- **地图自动检测**：自动读取游戏日志，检测当前所在地图
- **图片管理**：支持上传和查看地图图片
- **编辑模式**：自定义任务、地图和章节内容
- **主题切换**：深色/浅色主题

## 安装与启动

### 环境要求

- **操作系统**：Windows 10 / 11（64位）
- **运行环境**：无需额外安装 Node.js，使用打包后的便携版可直接运行
- **游戏要求**：已安装《流放之路2》客户端（用于地图自动检测功能）

### 启动方式

#### 方式一：源码运行（开发者）

1. 确保已安装 Node.js（推荐 v18+）
2. 在项目根目录打开终端，执行：
   ```bash
   npm install
   npm start
   ```

#### 方式二：便携版运行（普通用户）

1. 下载打包后的 `POE2 Map Guide.exe`
2. 双击运行即可

### 打包构建

如需自行构建便携版，执行：
```bash
npm run dist
```
打包后的文件位于 `dist/` 目录。

## 任务管理

### 勾选任务

- 点击任务左侧的复选框标记为已完成
- 已完成的任务会被添加删除线，颜色变淡
- 勾选状态会自动保存，下次打开保留

### 进度计算

- **章节进度**：该章节下所有任务的完成百分比
- **地图进度**：该地图下所有任务的完成百分比
- **总进度**：所有任务的完成百分比（顶部统计栏显示）

### 自动折叠

- 当章节或地图下所有任务都完成时，会自动折叠
- 手动点击章节/地图标题可随时展开或折叠
- 折叠状态在刷新页面后保持

## 章节与地图

### 章节结构

每个章节（Act）包含：
- **章节标题**：如"第一章：皆伐之影"
- **章节攻略**：整体开荒路线建议
- **章节Boss**：Boss打法攻略
- **地图列表**：该章节包含的所有地图
- **章节图片**：可查看/上传地图图片

### 地图信息

每个地图包含：
- **地图名称**：繁体中文 / 简体中文 双语显示
- **推荐等级**：地图推荐的角色等级
- **探索笔记**：详细的跑图攻略
- **任务列表**：该地图的所有任务条目
- **地图图片**：可查看/上传图片

## 编辑模式

### 进入编辑模式

点击章节标题旁的 **编辑** 按钮进入该章节的编辑模式。

### 编辑功能

| 操作 | 说明 |
|------|------|
| 编辑地图名 | 修改地图的名称（繁体/简体中文） |
| 编辑推荐等级 | 修改地图推荐等级 |
| 编辑任务 | 修改任务描述、优先级、奖励 |
| 添加任务 | 在当前地图添加新任务 |
| 删除任务 | 删除不需要的任务 |
| 拖拽排序 | 按住任务条目拖动调整顺序 |
| 添加图片 | 上传本地图片到章节/地图 |
| 删除图片 | 移除已上传的图片 |

### 任务编辑说明

- **优先级标签**：输入"主线"、"可选"或"事件"
- **奖励标签**：使用中文逗号分隔多个奖励，如"2天赋点, +10%冰冷抗性"
- **任务描述**：建议与游戏内任务名保持一致

### 添加章节

在最后一章下方，点击 **添加章节** 按钮可创建新章节。

### 保存机制

- 编辑内容自动保存到本地存储
- 可通过设置中的"重置地图数据"恢复默认数据
- 支持导出/导入缓存数据进行备份

## 浮窗模式

浮窗模式是一个始终置顶的小窗口，方便在游戏过程中查看当前地图任务。

### 开启浮窗

1. 点击顶部控制栏的 `📱 浮窗` 开关
2. 或使用快捷键（需在主窗口内）

### 浮窗界面

```
┌────────────────────────────┐
│ 🗕 🗖 🗙   🐾 地图名 / 地图名   │  ← 标题栏（可拖拽）
├────────────────────────────┤
│  ☐ 任务描述        [主线]   │
│  ☑ 任务描述        [可选]   │
│  ☐ 任务描述        [事件]   │
├────────────────────────────┤
│ ◀ 上一页    下一页 ▶       │
└────────────────────────────┘
```

### 浮窗控制

| 按钮 | 功能 |
|------|------|
| `🗕` | 折叠浮窗（只显示标题栏） |
| `🗖` | 展开浮窗 |
| `🗙` | 关闭浮窗 |
| `🎨` | 调整背景透明度（0%/20%/40%/60%/100%） |
| `A` | 调整字体大小 |
| `◀ ▶` | 切换上一个/下一个地图 |
| `拖拽标题栏` | 移动浮窗位置 |

### 浮窗特性

- **始终置顶**：浮窗不会被游戏窗口覆盖
- **点击穿透**：折叠状态下鼠标点击可穿透到游戏
- **透明度调节**：可调整背景透明度，不影响文字可读性
- **字体颜色**：支持自定义字体颜色（不影响标签颜色）
- **任务同步**：浮窗中勾选任务会同步到主窗口

一键清查功能用于快速查看所有**未完成且带有奖励**的任务，帮助玩家不错过任何重要奖励。

## 图片管理

### 查看图片

- 章节和地图支持显示多张图片
- 点击图片可进入大图查看模式
- 大图模式下支持：滚轮缩放、拖拽移动、点击关闭

### 上传图片

**编辑模式下**：
1. 点击章节/地图旁的 `📷 添加图片` 按钮
2. 选择本地图片文件（支持 PNG、JPG）
3. 图片会自动保存并显示

### 图片路径说明

- **缓存图片**：用户上传的图片，保存在应用数据目录
- **预设图片**：从 `data.json` 中加载的地图图片，位于 `image/POE2map/actMap/` 目录
- 优先显示缓存图片，没有缓存时显示预设图片

## 设置

点击顶部控制栏的 `⚙️ 设置` 按钮打开设置面板。

### 设置项目

#### 1. 游戏日志路径

- 添加 POE2 客户端日志文件路径以启用自动地图检测
- 默认自动检测常见安装路径
- 支持添加多个路径并切换活跃路径
- 日志文件通常位于：`游戏安装目录/logs/Client.txt`

#### 2. 缓存数据管理

- **导出缓存**：将当前的任务进度和编辑数据导出为 JSON 文件
- **导入缓存**：从 JSON 文件恢复任务进度和编辑数据
- **重置地图数据**：恢复默认地图数据（会丢失自定义修改）

#### 3. 主题

- 使用顶部 `☀️/🌙` 开关切换深色/浅色主题
- 主题偏好会自动保存

## 地图自动检测

### 工作原理

应用会监控 POE2 客户端日志文件（`Client.txt`），当玩家进入新地图时，日志会记录地图名称，应用解析后自动跳转到对应地图。

### 配置方法

1. 打开设置面板
2. 在"游戏日志路径"部分添加正确的日志文件路径
3. 确保路径被标记为"活跃"
4. 重新启动应用使配置生效

### 常见日志路径

```
G:\Program Files (x86)\Grinding Gear Games\Path of Exile 2\logs\Client.txt
C:\Program Files (x86)\Grinding Gear Games\Path of Exile 2\logs\Client.txt
D:\Program Files (x86)\Grinding Gear Games\Path of Exile 2\logs\Client.txt
```

### 自动检测效果

- 主窗口自动滚动到当前地图位置
- 浮窗自动切换显示当前地图的任务
- 浮窗标题显示当前地图名称（带自动检测标识）

## 快捷键与技巧

### 界面交互

| 操作 | 效果 |
|------|------|
| 点击章节标题 | 展开/折叠章节 |
| 点击地图标题 | 展开/折叠地图 |

### 浮窗技巧

- 折叠浮窗后可减少屏幕遮挡，需要时展开查看
- 调整透明度使浮窗与游戏画面融合
- 拖拽浮窗到屏幕边缘不遮挡游戏视野的位置

### 数据备份

- 定期使用"导出缓存"功能备份任务进度
- 导出的 JSON 文件可以复制到其他电脑导入
- 重装系统前务必导出缓存数据

### 编辑建议

- 编辑地图名时建议保持与游戏内地图名一致
- 添加奖励标签时统一格式，便于一键清查识别
- 利用拖拽排序将重要任务排在前面

## 常见问题

### Q: 应用启动后数据没有加载？

A: 检查 `data.json` 文件是否存在且格式正确。尝试点击设置中的"重置地图数据"恢复默认。

### Q: 地图自动检测不工作？

A: 确认日志路径配置正确，且游戏正在运行。某些情况下需要以管理员权限运行应用。

### Q: 浮窗无法显示在游戏上方？

A: 部分全屏游戏可能覆盖浮窗。尝试将游戏切换为窗口化或无边框窗口模式。

### Q: 编辑后的内容丢失了？

A: 确保应用正常关闭（不要强制结束进程）。如仍丢失，检查磁盘空间是否充足。

### Q: 图片上传后无法显示？

A: 确认图片格式为 PNG 或 JPG。检查应用数据目录是否有写入权限。
